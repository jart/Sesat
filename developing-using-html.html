Developing using Html

General information

We use XHTML 1.0 with the TRANSITIONAL doctype. Our goal should be to use the STRICT doctype, but with the extensivey use of javascript today it would be impossible to make it valid.

 The html code is generated mostly with velocitytemplates, but the structure of the page is done in a jsp file. In this file different structure element is defined. In views.xml each tab is declared with the spesific templates it's going to use. The templates are inherited from projects above, but can be overriden if necessary.

Guidelines

Guidelines we should try to follow:

  • All element- and attribute names must be in lower case. All attribute values must be quoted.
  • Close all elements. Example: <img src="image.jpg" alt="" />, <br />
  • Don't use deprecated elements and attributes  http://www.html-reference.com/depreciated.htm
  • Avoid using tables for layout
  • Keep number of elements as low as possible (overuse of div's)
  • Allways add alt-attribute in <img> element
  • Allways set width and height attributes for images inline. Example: <img src="image.jpg" width="100" height="200" alt="" />
  • Encode ampersands in url's
  • Use relative path instead of absolute where this is possible
  • Make javascript unobtrusive where it's possible

www subdomain - we should use http://sesam.no instead of http://www.sesam.no.
All internal, and external links, should use the same way to link to sesam, not mixing the two.

The benefits by doing this is:

  • Simpler development and maintenance
  • Compatibility with futures web browsers
  • Faster download and rendering of pages
  • Better accessibility (for screenreaders and alternative browsing)
